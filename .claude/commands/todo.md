할 일을 관리합니다. `docs/TODO.md` 파일에 기록하고 업데이트합니다.

## 사용법

- `/todo 추가: <할 일>` — 새 항목 추가 (실행 가능 여부 자동 판단)
- `/todo 완료: <항목 내용>` — 항목을 완료로 처리
- `/todo 보류: <항목 내용> / 이유: <이유>` — 항목을 보류로 처리
- `/todo 현황` — 전체 현황 요약 출력

## 실행

아래 지침에 따라 `docs/TODO.md` 를 직접 읽고 수정하세요.

$ARGUMENTS

### 행동 지침

1. 먼저 `docs/TODO.md` 파일을 읽어 현재 상태를 파악한다.
2. 입력된 명령에 따라 아래 규칙을 적용한다.

**추가**:
- 항목의 실행 가능 여부를 판단한다.
  - 실행 가능 → `## 📋 대기 중` 섹션에 `- [ ] 항목` 추가
  - 선행 조건 필요 / 정보 부족 → `## 🚧 보류` 섹션에 `- [ ] 항목 _(보류: 이유)_` 추가
- 판단 근거를 한 줄로 출력한다.

**완료**:
- 해당 항목을 `- [x] 항목` 으로 변경하고 `## ✅ 완료` 섹션으로 이동한다.

**보류**:
- 해당 항목을 `## 🚧 보류` 섹션으로 이동하고 `_(보류: 이유)_` 를 덧붙인다.

**현황**:
- 각 섹션별 항목 수와 목록을 출력한다.

3. `> 마지막 업데이트:` 날짜를 오늘 날짜로 갱신한다.
4. 파일의 섹션 구조(헤더, 구분선, 주석)는 절대 변경하지 않는다.
