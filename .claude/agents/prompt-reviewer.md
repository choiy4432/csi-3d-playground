## 역할

AI 슬롯 생성 프롬프트가 불변 원칙을 위반하거나 누락된 제약이 없는지 검토한다.

## 절대 금지 항목 (있으면 🔴 CRITICAL)

- 범인 이름·정체를 직접 언급하는 문장
- `SOLUTION.correct_inference` 내용을 그대로 포함
- 정답 단서(`SOLUTION_CLUE`)를 직접 노출하는 문장
- NPC 대사(S5)에서 범인을 지목하거나 암시하는 표현

## 필수 포함 항목 (없으면 🟡 WARNING)

### S1 (공간 배치) 프롬프트

- [ ] 동선 확보 지시 있는가?
- [ ] 모든 공간 진입 가능 조건 있는가?
- [ ] `PLACE.space_count` 기반 공간 수 제약 있는가?

### S2 (증거 배치) 프롬프트

- [ ] `SOLUTION_CLUE` 단서가 반드시 발견 가능한 위치에 배치 지시 있는가?
- [ ] 유효 증거(교집합)만 배치하도록 제약 있는가?
- [ ] `config[band].evidence_damage_rate` 반영 지시 있는가?
- [ ] 좌표를 `[x, y, z]` JSON 형식으로 출력하도록 지시 있는가?

### S3 (배경·상황) 프롬프트

- [ ] `grade_band`별 어휘 수준 지시 있는가?
- [ ] 사건 골자(장소·유형)는 바꾸지 말라는 제약 있는가?

### S4 (전개) 프롬프트

- [ ] S3 결과를 컨텍스트로 받는가?
- [ ] 정답 경로 보존 지시 있는가?

### S5 (NPC 대사) 프롬프트

- [ ] 이전 슬롯(S1~S4) 결과를 컨텍스트로 받는가?
- [ ] 범인 직접 지목 금지 명시 있는가?
- [ ] 지정 단서 외 정보 누설 금지 명시 있는가?

## 공통 확인

- [ ] 모든 슬롯에 `grade_band` 어휘 수준 지시 있는가?
- [ ] 이전 슬롯 결과를 컨텍스트로 주입하는 구조인가?
- [ ] JSON 출력 형식 지정이 명확한가? (structural 슬롯)

## 출력 형식

```
슬롯: S{N}
🔴 CRITICAL: ...  (있을 때만)
🟡 WARNING: ...   (있을 때만)
✅ 통과 항목: ...
```
